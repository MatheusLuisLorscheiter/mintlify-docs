# Autenticação

A API do Sistema de Representantes utiliza autenticação baseada em API Key para garantir acesso seguro aos dados.

## Obtendo uma API Key

Para obter uma API Key:

1. Acesse o painel administrativo do sistema
2. Navegue até **Configurações > Integrações > API Keys**
3. Clique em **Gerar Nova API Key**
4. Copie e armazene sua chave com segurança

<Warning>
**Importante**: A API Key será exibida apenas uma vez. Certifique-se de copiá-la e armazená-la em um local seguro.
</Warning>

## Usando a API Key

Inclua sua API Key no header `x-api-key` em todas as requisições:

```bash
curl -X GET \
  https://api.sistema-representantes.com.br/api/v1/clients \
  -H 'x-api-key: sua-chave-api-aqui'
```

## Exemplo com diferentes linguagens

<CodeGroup>

```javascript Node.js
const response = await fetch('https://api.sistema-representantes.com.br/api/v1/clients', {
  headers: {
    'x-api-key': 'sua-chave-api-aqui'
  }
});

const data = await response.json();
```

```python Python
import requests

headers = {
    'x-api-key': 'sua-chave-api-aqui'
}

response = requests.get(
    'https://api.sistema-representantes.com.br/api/v1/clients',
    headers=headers
)

data = response.json()
```

```php PHP
$ch = curl_init();

curl_setopt($ch, CURLOPT_URL, 'https://api.sistema-representantes.com.br/api/v1/clients');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'x-api-key: sua-chave-api-aqui'
]);

$response = curl_exec($ch);
$data = json_decode($response, true);

curl_close($ch);
```

</CodeGroup>

## Erros de Autenticação

| Código | Mensagem | Descrição |
|--------|----------|-----------|
| 401 | API key ausente | O header `x-api-key` não foi incluído na requisição |
| 401 | API key inválida | A API key fornecida não existe ou está incorreta |
| 403 | Empresa inativa | A empresa associada à API key está inativa |

## Segurança

### Boas Práticas

- **Nunca exponha sua API Key** em código client-side ou repositórios públicos
- **Use HTTPS** em todas as requisições
- **Rotacione suas chaves** regularmente
- **Limite o acesso** apenas aos IPs necessários (quando disponível)

### Variáveis de Ambiente

Recomendamos armazenar sua API Key em variáveis de ambiente:

```bash
# .env
API_KEY=sua-chave-api-aqui
```

```javascript
// Uso no código
const apiKey = process.env.API_KEY;
```

## Limites e Quotas

- **Rate Limit**: 1000 requisições por hora por API key
- **Concurrent Requests**: Máximo de 10 requisições simultâneas
- **Payload Size**: Máximo de 5MB por requisição

Se você exceder esses limites, receberá um erro 429 (Too Many Requests).

# Introdução à API

A API REST do Sistema de Representantes fornece acesso programático às principais funcionalidades do sistema.

## Endpoints Disponíveis

### Clientes
- `GET /api/v1/clients` - Listar clientes
- `POST /api/v1/clients` - Criar novo cliente

### Vendas
- `GET /api/v1/sales` - Listar vendas
- `POST /api/v1/sales` - Criar nova venda

## Formato das Respostas

Todas as respostas seguem um formato padronizado:

### Sucesso
```json
{
  "data": { ... },
  "page": 1,
  "limit": 20,
  "total": 100
}
```

### Erro
```json
{
  "error": "Descrição do erro"
}
```

## Headers Obrigatórios

| Header | Valor | Descrição |
|--------|-------|-----------|
| `x-api-key` | string | Sua chave de API para autenticação |
| `Content-Type` | `application/json` | Tipo de conteúdo (para POST) |

## Limites de Rate

- 1000 requisições por hora por API key
- Máximo de 100 registros por página nas listagens

## Paginação

Os endpoints de listagem suportam paginação através dos parâmetros:

- `page` - Número da página (padrão: 1)
- `limit` - Itens por página (padrão: 20, máximo: 100)

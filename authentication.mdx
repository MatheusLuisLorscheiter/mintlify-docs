# Autenticação

A API usa autenticação baseada em API Key para garantir acesso seguro aos dados.

## Como funciona

Todas as requisições devem incluir o header `x-api-key`:

```bash
x-api-key: sua-api-key-aqui
```

## Exemplo de uso

<CodeGroup>

```bash cURL
curl -X GET https://api.sistema-representantes.com.br/api/v1/clients \
  -H "x-api-key: sua-api-key-aqui"
```

```javascript JavaScript
const response = await fetch('https://api.sistema-representantes.com.br/api/v1/clients', {
  headers: {
    'x-api-key': 'sua-api-key-aqui'
  }
});
```

```python Python
import requests

headers = {
    'x-api-key': 'sua-api-key-aqui'
}

response = requests.get(
    'https://api.sistema-representantes.com.br/api/v1/clients',
    headers=headers
)
```

</CodeGroup>

## Obter uma API Key

1. Acesse o painel administrativo
2. Navegue até **Configurações** → **Integrações** → **API Keys**
3. Clique em **Gerar Nova API Key**
4. Copie e armazene sua chave com segurança

<Warning>
A API Key será exibida apenas uma vez. Se perder, será necessário gerar uma nova.
</Warning>

## Segurança

### Boas práticas

- **Nunca exponha** sua API Key em código público
- **Use HTTPS** sempre
- **Armazene em variáveis de ambiente**
- **Rotacione** suas chaves periodicamente

### Exemplo com variáveis de ambiente

```javascript .env
API_KEY=sua-api-key-aqui
```

```javascript
const apiKey = process.env.API_KEY;

const response = await fetch(url, {
  headers: {
    'x-api-key': apiKey
  }
});
```

## Erros de autenticação

| Código | Mensagem | Descrição |
|--------|----------|-----------|
| `401` | `API key ausente` | Header x-api-key não foi enviado |
| `401` | `API key inválida` | API key não existe ou está incorreta |
| `403` | `Empresa inativa` | A empresa está inativa no sistema |
